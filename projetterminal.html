En informatique, un terminal est une variété de périphérique réseau placé à l'extrémité d'un nœud. Le terminal est un point d’accès de communication entre l'homme, la machine, ou un réseau de machine. Dans un réseau décentralisé de type client-serveur il correspond aujourd'hui à l'association minimale d'un périphérique de saisie (clavier) ou de pointage (souris) et d'un moniteur reliés à l'unité centrale. Sous Unix et en génie logiciel, il désigne par abus de langage une fenêtre d'invite de commande contenant un shell Unix.


[root@localhost ~]# le Manuel (terminal mode)

[root@localhost ~]# le Manuel (terminal mode) /man

La commande man est employée pour vous montrer le manuel des autres commandes. (man = manuel) Pour obtenir une bonne compréhension, Pour quitter man, appuyez sur la touche q
Certains utilitaires sont mieux documentés en tant que texte info. par exemple essayez

[root@localhost ~]# info man
On peut également avoir une aide rapide sur la plupart des commandes par l'option –help (ou parfois -h)

[root@localhost ~]# man --help

[root@localhost ~]# man -h
..


[root@localhost ~]# Commande ls

[root@localhost ~]# Commande ls / ls

La commande ls liste les fichiers et les sous-dossiers. (ls = list)
[root@localhost ~]# ls

Pour obtenir plus d'information, utilisez l'option -l (pour version longue) :

[root@localhost ~]# ls -l

Pour afficher aussi les fichiers "cachés" (en fait ceux qui commencent par un point) utilisez l'option -A (pour All) :

[root@localhost ~]# ls -A
..


[root@localhost ~]# Commande cd

[root@localhost ~]# Commande cd / cd

La commande cd vous permettra de changer de répertoire (cd = change directory). Quand vous ouvrez un terminal en mode utilisateur vous serez dans votre répertoire local (/home/utilisateur). Dans un système linux la référence au fichier s'appelle un chemin. Dans un chemin le nom des répertoires et des fichiers sont séparés par un "/". Il existe deux types de chemin : absolu et relatif.

[root@localhost ~]# cd /home/utilisateur/dossier

Le symbole "~" peut remplacer le chemin absolu vers votre répertoire personnel soit "/home/utilisateur/".

[root@localhost ~]# cd ~/Desktop
..

[root@localhost ~]# Commande apt-get

[root@localhost ~]# Commande apt-get / apt-get

La commande apt-get permet de gérer les paquets Debian de votre Ubuntu. Exemple :

[root@localhost ~]# sudo apt-get update

permet de mettre à jour la liste des paquets disponibles, commande à taper en premier avant toute installation pour être sûr d'avoir les mises à jour.
[root@localhost ~]# sudo apt-get upgrade

permet de mettre à jour les paquets déjà installés, à taper pour faire les mises à jour de sécurité.

[root@localhost ~]# sudo apt-get install soft

Installera le logiciel "soft" en gérant les dépendances, donc "apt" vous demandera peut être d'installer d'autres paquets en complément.

[root@localhost ~]# sudo apt-get remove soft

désinstallera le paquet "soft".

[root@localhost ~]# sudo apt-get autoremove soft

désinstallera "proprement" le paquet "soft" ainsi que ses dépendances
